Class AbstractMessaging

Abstract Messaging implementation

description

There can be different messaging use cases where different type of Subjects can be used i.e. {Rx.Subject} or {Rx.ReplaySubject} which determines what behavior is required for the use case.

example:
 public class TestMessaging extends AbstractMessaging {
  static fromSubjects(subjects: any): AppMessaging {
     return new ContribMessaging(subjects);
  }

  constructor(subjects?: any) {
      super(subjects);
  }

  public newSubject<T>(): Subject<T> {
      return new ReplaySubject<T>();
  }
 }
